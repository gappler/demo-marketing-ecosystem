# Yowie Content Engine Design

## Overview

An interactive HTML tool that demonstrates Yowie's content generation pipeline: structured campaign brief in, on-brand segment-specific channel-ready content out. The tool reads from the Brand Configuration Layer (brand_config.json) and uses pre-generated AI-written copy embedded at build time.

Content is generated by Claude Code during build, not live. The tool is a showcase of the system logic with real quality output. Labeled clearly: "Content generated by Claude Code from the Brand Configuration Layer and campaign brief."

## Approach

Single self-contained HTML file with embedded JSON data — same pattern as brand_config_viewer.html. All campaign briefs, generated copy variants, rationale, and rendering logic in one portable file.

## Data Structure

### Briefs

Array of 4 campaign brief objects, each containing:

- `id`, `name`, `description`
- Brief inputs: `product_id`, `product_line`, `audience_segments[]`, `channels[]`, `campaign_stage`, `primary_message`, `secondary_messages[]`, `pricing`, `proof_points[]`, `tone_override` (optional), `cta`, `personalization_variables[]`

### Generated Content

Keyed by `brief_id`, organized by output mode:

- `single` — one featured piece (specific audience + channel)
- `segment_variants` — same channel, different audiences, with annotations on what changed and why
- `channel_variants` — same audience, different channels, with annotations on register shifts
- `campaign_package` — brief summary at top, full audience x channel matrix, campaign metadata (suggested timing, content hierarchy)

Each piece of copy includes: the copy itself, voice rules applied, messaging pillar used, audience adaptation, and a short rationale.

## Output Modes

### Single Item
One content card with copy + metadata sidebar showing voice rules applied, pillar used, audience adaptation, and rationale.

### Segment Variants
Side-by-side cards (e.g., backcountry vs. casual) with copy + annotations highlighting what changed between variants and why.

### Channel Variants
Stacked cards (email, social, landing page/product page) styled to hint at channel format, with annotations on register shifts.

### Full Campaign Package
Brief summary at top, then grid of all segment x channel combinations as expandable cards.

## Content Card Anatomy

Each generated content card contains:

1. **Channel + audience label** (e.g., "Email → Backcountry Hikers")
2. **The copy** — monospace font, distinct container with subtle left border accent
3. **Rules applied** — collapsible section listing voice rules, channel register, audience adaptation
4. **Pillar used** — primary messaging pillar with audience-specific resonance line
5. **Rationale** — 1-2 sentences explaining why this content was shaped this way

Segment and channel variant modes include **"What changed"** annotations between cards.

## UI Layout

### Header
"Yowie Content Engine" with subtitle attribution line.

### Left Sidebar (Fixed)
- 4 brief cards: name, product, campaign stage badge
- Active brief highlighted
- Output mode selector: 4 buttons (Single, Segments, Channels, Full Package)
- Each brief has a default mode but all modes work with all briefs

### Main Content Area
- **Brief summary panel** — collapsible card showing all brief inputs. Always visible in Full Package mode.
- **Content output** — rendered based on selected mode

## Visual Style

Same dark theme as brand_config_viewer.html:
- Dark navy background (`#0a0a1a`)
- Gold accents (`#e2c87d`) for headings
- Teal highlights (`#64ffda`) for interactive elements
- Monospace font for generated copy to distinguish from UI chrome

## Build Process

1. Pre-generate all content: write 4 briefs, generate AI copy for every audience x channel combination with rationale and rule annotations
2. Embed everything as JSON in a single HTML file
3. Build the interactive UI with sidebar brief selector, mode switcher, and content rendering
4. Match the brand config viewer's dark theme
