# Yowie Research & Brief Builder Design

## Overview

An interactive HTML tool that demonstrates the upstream intelligence feed for Yowie's content pipeline. It takes raw inputs (product data, audience research, competitive landscape, seasonal context), performs analysis, and produces structured campaign briefs that feed directly into the Content Engine. The tool makes the human decision point explicit: AI produces drafts, humans make decisions.

Content is pre-generated by Claude Code during build, not live. Three example briefs with full research, reasoning chains, and output briefs are embedded as JSON. Labeled clearly with attribution.

## Approach

Single self-contained HTML file with embedded JSON data — same pattern as brand_config_viewer.html and content_engine.html. All research data, reasoning chains, output briefs, and rendering logic in one portable file.

## Data Structure

### `pipeline`

Metadata for the pipeline diagram. 5 stages:

1. **Raw Inputs** — product data, market research, seasonal context. Color: teal.
2. **AI Research & Analysis** — competitive landscape, audience analysis, positioning. Color: teal.
3. **Draft Brief** — structured campaign brief generated from research. Color: teal.
4. **Human Review** — the decision gate. Color: gold. Includes a review checklist:
   - Does the primary message match the positioning angle?
   - Are the proof points verifiable and specific?
   - Is the channel mix right for this audience's media habits?
   - Does the CTA match the campaign objective?
   - Any tone overrides needed?
5. **Content Engine** — approved brief feeds downstream content generation. Color: teal.

### `briefs`

Array of 3 brief builder objects. Each contains:

- `id`, `name`, `description`, `product_id`, `campaign_objective`
- `research`:
  - `competitive_landscape`: array of competitor objects (name, price_range, positioning, strengths, weaknesses)
  - `audience_analysis`: object with profile, pain_points (array), buying_triggers (array), media_habits (array)
  - `seasonal_context`: string
  - `positioning_angle`: string — Yowie's specific angle
- `reasoning`:
  - `message_rationale`: string — why this primary message
  - `channel_rationale`: string — why these channels
  - `proof_point_rationale`: string — why these proof points
  - `pillar_rationale`: string — why this pillar leads
- `output_brief`: object in Content Engine format (product_id, product_line, audience_segments, channels, campaign_stage, primary_message, secondary_messages, pricing, proof_points, tone_override, cta, personalization_variables)

### The Three Briefs

**Brief 1: Slab 35QT Cooler Acquisition** (`slab_acquisition`)
- Campaign objective: new customer acquisition in the cooler market
- Research: YETI Tundra 35 ($300-350, lifestyle brand premium positioning), Stanley (viral social momentum, fashion-adjacent), RTIC 35QT ($200, "same as YETI for less"), Igloo BMX 25 ($40-60, budget/disposable)
- Audience: casual car campers — price-conscious but quality-aware, tired of replacing cheap coolers, skeptical of premium brand markups
- Positioning angle: Yowie slots between RTIC's "cheaper alternative" and YETI's build quality — same rotomolded construction, no lifestyle tax, doesn't position as a knockoff
- Output: acquisition campaign, social/email/landing page channels, "pay for the gear" pillar

**Brief 2: Lowline 2P for Cold-Weather Specialists** (`lowline_cold_weather`)
- Campaign objective: product launch to cold-weather niche
- Research: MSR Access series ($500-700, ski-touring focus), Hilleberg Anjan 2 ($700+, expedition-grade), Big Agnes Copper Spur ($400-450, ultralight priority over weather), REI co-op tents ($250-350, generalist)
- Audience: cold-weather specialists — conditions-driven, most spec-literate, trust lab testing over marketing
- Positioning angle: purpose-built low-profile design vs. competitors who either over-build (Hilleberg price) or under-build (Big Agnes wind performance). The 40" profile is a deliberate engineering choice, not a weight-saving compromise
- Output: product launch, product page/email/social channels, "built for the job" pillar

**Brief 3: Foldout Soft Cooler Cross-Pollination** (`foldout_cross_sell`)
- Campaign objective: cross-sell to existing Slab buyers (Phase 3 strategy)
- Research: YETI Hopper ($250-350, premium soft cooler), Hydro Flask Day Escape ($150, lifestyle-adjacent), Coleman soft coolers ($25-40, disposable), Engel HD30 ($130, fishing/marine focus)
- Audience: existing Slab 35QT buyers — already trust Yowie, this is the everyday-use cooler they didn't know they needed
- Positioning angle: "you bought the Slab for camping, this is the one for everything else." Not competing on ice retention (24hr vs 5-day), competing on versatility and build quality vs. disposable alternatives
- Output: cross-sell/retention campaign, email/social channels, "built for the job" pillar, personalization referencing Slab purchase

## Pipeline Diagram

Horizontal flowchart at the top of the tool, spanning full width above the sidebar/main layout. Approximately 80px tall.

5 stages connected by arrows:
- AI-driven stages (Raw Inputs, Research, Draft Brief, Content Engine): teal accents
- Human decision stage (Human Review): gold accents, visually distinct border/style, "decision gate" indicator
- Label below: "AI produces drafts. Humans make decisions."

Clicking a stage scrolls the main content to the corresponding section and highlights it briefly. The Human Review stage, when clicked, shows the review checklist.

## UI Layout

### Pipeline (fixed at top)
Horizontal flowchart, full width, compact (~80px). Clickable stages scroll to sections below.

### Left Sidebar (fixed, below pipeline)
- 3 brief cards: name, product, campaign objective badge
- Active brief highlighted with gold left border
- Attribution footer

### Main Content Area (scrollable, below pipeline)
Sections correspond to pipeline stages with scroll anchors:

**Research & Analysis section:**
- Competitive landscape as a comparison grid — competitors in columns with price, positioning, strengths, weaknesses
- Audience analysis — profile, pain points, buying triggers, media habits
- Seasonal context
- Positioning angle — visually called out

**Reasoning section:**
- Collapsible cards for each reasoning chain: message, channel, proof point, pillar rationale
- Each card shows "Research said X → Brief decision: Y"

**Draft Brief section:**
- Structured brief as a read-only form-like card (same field layout as Content Engine brief summary)
- "Content Engine compatible" badge

**Human Review section:**
- Review checklist with checkbox-style items (visual only, not interactive)
- Note: "In production, this is where a marketing lead reviews the AI-generated brief before approving it for content generation."

## Visual Style

Same dark theme as brand_config_viewer.html and content_engine.html:
- Dark navy background (#0a0a1a)
- Gold accents (#e2c87d) for headings and human decision elements
- Teal highlights (#64ffda) for AI-driven elements and interactive states
- Monospace font for brief output fields

## Build Process

1. Pre-generate all research data, reasoning chains, and output briefs for 3 examples
2. Embed as JSON in a single HTML file
3. Build pipeline diagram with clickable scroll-to behavior
4. Build sidebar with brief card switching
5. Build main content sections with research grids, reasoning cards, brief display, and review checklist
6. Match existing tool dark theme
